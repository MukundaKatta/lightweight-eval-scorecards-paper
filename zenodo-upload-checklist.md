# Zenodo Upload Checklist

This is the recommended first public posting route for the current manuscript package.

## Files to include

Upload these files together:

- `paper.md`
- `abstract.txt`
- `keywords.txt`
- `author-bio.txt`
- `cover-note.txt`
- `paper.bib`
- `submission-metadata.json`
- `assets/workflow-figure.svg`

## Suggested Zenodo form values

### Title

`Lightweight Evaluation and Operational Scorecards for Tool-Using AI Agents`

### Upload type

`Publication`

### Publication type

`Technical note` or `Preprint`

If both are available and you want the more research-shaped label, choose `Preprint`.

### Authors

`Mukunda Rao Katta`

### Description

Use the contents of `abstract.txt` as the base description. If Zenodo allows a slightly longer description, add one short sentence that points to the public artifact family behind the paper.

Suggested extra sentence:

`The manuscript is accompanied by public evaluation datasets, interactive demo apps, and portfolio artifacts that illustrate the workflow described in the paper.`

### Keywords

Use the list from `keywords.txt`.

### License

If you want the least friction for public sharing, use:

`Creative Commons Attribution 4.0 International (CC BY 4.0)`

If you want a more conservative all-rights-reserved route for the text, review Zenodo's available publication licenses and pick the one that matches your preference before publishing.

## Submission flow

1. Sign in to Zenodo.
2. Start a new upload.
3. Upload the files listed above.
4. Enter the title exactly as written.
5. Add your author name.
6. Paste the abstract into the description field.
7. Add the keywords.
8. Choose the upload type and publication type.
9. Choose the license.
10. Review the metadata one more time.
11. Publish.

## Final pre-publish checks

- make sure the manuscript title matches the Zenodo title exactly
- make sure your name is spelled the same way everywhere
- make sure the abstract on Zenodo matches `abstract.txt`
- make sure the workflow figure is included in the upload
- make sure the paper only describes artifacts that are already public

## After publishing

Once the Zenodo record is live:

1. save the DOI and public URL
2. add that DOI to your GitHub, GitLab, and Bitbucket profile READMEs
3. add the DOI to your Codeberg Pages portfolio
4. reuse the same manuscript package for OSF Preprints or SSRN
